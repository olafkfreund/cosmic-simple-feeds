## Build Commands

This project uses `just` as a task runner (recipes in `justfile`):

```bash
just                    # Default: builds release
just build-debug        # Debug build
just build-release      # Release build
just check              # Clippy with pedantic warnings
just run                # Run with RUST_BACKTRACE=full (release mode)
just install            # Install binary + desktop entry + icons to /usr/
just uninstall          # Remove installed files
just clean              # cargo clean
```

The project requires `libcosmic` as a git submodule. After cloning:
```bash
git submodule update --init --recursive
```

System library dependencies: `dbus`, `wayland`, `libxkbcommon`, `openssl` development headers.

## Architecture

The codebase is six files in `src/`:

- **`main.rs`** - Entry point. Loads `cosmic-config`, initializes i18n, launches the applet event loop via `cosmic::applet::run::<AppModel>()`.
- **`app.rs`** - Core application logic implementing `cosmic::Application` for `AppModel`. Contains the Elm-architecture message loop (`Message` enum), popup management, feed CRUD, and the `save_feeds()` helper. The popup view is built inline via a closure passed to `app_popup()`.
- **`config.rs`** - `Config` struct with `CosmicConfigEntry` derive macro. Two fields: `feeds: Vec<String>` and `refresh_interval_seconds: u64`. Versioned (currently v3), persisted via `cosmic-config`.
- **`feed.rs`** - Feed fetching (`fetch_all`), URL validation (`validate_url`), HTTP client setup, and text sanitization (`sanitize_text`). Includes 16 unit tests.
- **`error.rs`** - `FeedError` enum with `thiserror` derives.
- **`i18n.rs`** - Fluent-based localization boilerplate. Translations live in `i18n/<lang>/feeds.ftl`. Exposes the `fl!()` macro.

### Key Patterns

- **Applet popup model**: The applet shows a "Feeds" button in the panel. Clicking toggles a Wayland popup (`app_popup`/`destroy_popup`). The popup content is rendered via a closure that captures `&AppModel` state.
- **Multi-feed fetching**: `fetch_all()` uses `futures_util::future::join_all` to fetch all feeds in parallel. Partial failures are logged but don't prevent other feeds from loading.
- **Config persistence**: Adding/removing feeds writes to `cosmic-config` immediately via `save_feeds()` which calls `set_feeds()` (generated by the `CosmicConfigEntry` derive).
- **State machine**: `State` enum tracks `Loading`/`Loaded`/`Error(FeedError)`.
- **Background refresh**: Periodic refresh keeps old items visible instead of showing a loading state.

### Resources

- `resources/app.desktop` - Desktop entry with `X-CosmicApplet=true` and `NoDisplay=true`
- `resources/app.metainfo.xml` - AppStream metadata
- `resources/icons/` - SVG icons (color + symbolic variants)
- `libcosmic/` - Git submodule pointing to `pop-os/libcosmic`

### Localization

Fluent files in `i18n/<lang>/feeds.ftl`. Fallback language is English. Use the `fl!("key")` macro for translated strings.

### Naming Convention

This project follows the [COSMIC trademark policy](https://github.com/pop-os/cosmic-epoch/blob/master/TRADEMARK.md):
- Package name uses `cosmic-ext-` prefix (third-party convention)
- App ID uses `io.github.` reverse-domain prefix
- App ID (D-Bus/dots): `io.github.olafkfreund.cosmic-ext-applet-feeds`
- App ID (filesystem/dashes): `io.github-olafkfreund.cosmic-ext-applet-feeds`
