## What This Is

A COSMIC Desktop panel applet that fetches and displays RSS feed items in a popup. It uses `libcosmic` (iced-based) for UI and `reqwest` + `rss` for network/parsing. The binary is called `feeds` and registers as `com.github.marcossl10.cosmic-simple-feeds`.

## Build Commands

This project uses `just` as a task runner (recipes in `justfile`):

```bash
just                    # Default: builds release
just build-debug        # Debug build
just build-release      # Release build
just check              # Clippy with pedantic warnings
just run                # Run with RUST_BACKTRACE=full (release mode)
just install            # Install binary + desktop entry + icons to /usr/
just uninstall          # Remove installed files
just clean              # cargo clean
```

The project requires `libcosmic` as a git submodule. After cloning:
```bash
git submodule update --init --recursive
```

System library dependencies: `dbus` and `gtk3` development headers.

## Architecture

The codebase is four files in `src/`:

- **`main.rs`** - Entry point. Loads `cosmic-config`, initializes i18n, launches the applet event loop via `cosmic::applet::run::<AppModel>()`.
- **`app.rs`** - Core application logic implementing `cosmic::Application` for `AppModel`. Contains the Elm-architecture message loop (`Message` enum), popup management, feed CRUD, and the async `fetch_feed()` helper. The popup view is built inline via a closure passed to `app_popup()`.
- **`config.rs`** - `Config` struct with `CosmicConfigEntry` derive macro. Two fields: `feeds: Vec<String>` and `refresh_interval_seconds: u64`. Versioned (currently v3), persisted via `cosmic-config`.
- **`i18n.rs`** - Fluent-based localization boilerplate. Translations live in `i18n/<lang>/feeds.ftl`. Exposes the `fl!()` macro.

### Key Patterns

- **Applet popup model**: The applet shows a "News" button in the panel. Clicking toggles a Wayland popup (`app_popup`/`destroy_popup`). The popup content is rendered via a closure that captures `&AppModel` state.
- **Feed fetching**: `fetch_feed()` returns a `Task<Message>` using `Task::perform` with an async block. Only the first feed URL in the list is fetched at a time. A `time::every` subscription triggers `Message::Refresh` periodically.
- **Config persistence**: Adding/removing feeds writes to `cosmic-config` immediately via `set_feeds()` (generated by the `CosmicConfigEntry` derive).
- **State machine**: `State` enum tracks `Loading`/`Loaded`/`Error(String)`.

### Resources

- `resources/app.desktop` - Desktop entry with `X-CosmicApplet=true` and `NoDisplay=true`
- `resources/app.metainfo.xml` - AppStream metadata
- `resources/icons/` - SVG icons (color + symbolic variants)
- `libcosmic/` - Git submodule pointing to `pop-os/libcosmic`

### Localization

Fluent files in `i18n/<lang>/feeds.ftl`. Fallback language is English. Use the `fl!("key")` macro for translated strings (though the current UI mostly uses hardcoded Portuguese strings).

## Notes

- `Cargo.lock` is gitignored
- The app ID in `justfile` (`com.github-marcossl10.cosmic-simple-feeds`) uses a hyphen, while `app.rs` uses a dot (`com.github.marcossl10.cosmic-simple-feeds`) - these are intentionally different (filesystem vs D-Bus conventions)
- To test with a local `libcosmic` checkout, uncomment the `[patch]` section in `Cargo.toml`
- Rust edition is 2024
